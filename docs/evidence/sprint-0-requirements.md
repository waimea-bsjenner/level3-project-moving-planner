# Project Requirements

## Identified Problem or Need

I recently moved house and had a hard time finding specific kitchen equipment to cook a food I was craving. It is hard to remember what has been put into what box, and often people will be helping, so you might not even know where a specific item even is.


## End-User Requirements

My typical end user will be moving house and have stuff to take with them to a new house. They will have put their stuff in boxes, or some form of container


## Proposed Solution

I intend to create an app that can have multiple users, and be used multiple times. It will have the user enter every item and what box they are putting it into so that when they start unpacking, they can find specific items faster.


# Relevant Implications

## Functionality

The end product should have minimal bugs, glitches, and exploits, and it should work on different devices (mobile, laptop, etc)

### Relevance to the System

This implication is relevant because if the app is constantly glitching, then it will be harder to use and take longer to use effectively, or it might just not work at points. This will be a considerable time loss, even so to the point where it might be more effecient to just write the contents of the box on the box itself.

### Impact / Considerations

the project will need to prioritize bug fixing, and stability of the app, so it is functional for extended periods of time.



## Usability

The end product should be easy and intuitive to use, and allow for easy mistake correcting and editing

### Relevance to the System

If the end product is hard to use, then it will take significantly longer to use the app, and time is important when it comes to packing up things, as often the move out date will be the next day for most people.

### Impact / Considerations

the project will need to be well designed and intuitive, with the use of appropriately colored buttons, links, etc, and have a flow that feels natural to use.



## Accessbility

the end product should be accessible for different devices, and people with different disabilities

### Relevance to the System

to use this app efficiently, it will be needed to be done on a handheld device you can carry with you as you pack up boxes, for example phone, laptop, tablets, as having to go to a desktop back and forth as you enter items will waste significant time.

### Impact / Considerations

The project will need to be usable for different screen ratios, so I will have to design the app around this, and also include features for people with disabilities such as colorblindness



## Privacy

the end product should keep data safe from people who shouldn't have access

### Relevance to the System

If you have people helping you with a move, then its likely they will also be using the app to help pack things up, however if you don't want them to help with the next move, then there should be a way to only give them access to the moves you had them help you with.

### Impact / Considerations

the project will need to have a way to allow limited access to other users who are simply helping with a move, but aren't signed in



## Future proofing

the end product should be usable in a few years time, as it is intended for the use of multiple moves for multiple users

### Relevance to the System

If the end user move several times, over however many years, this app should be functional for all of those times, as if it isn't built for long-term use, then there could be issues with using the app that mess up plans the end user might've had.

### Impact / Considerations

the project will need to take long term use into account, including things such as help for remembering passwords, and long term data storage.
